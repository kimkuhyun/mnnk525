# ADR 005 — 뉴스 회사 매칭 단계 제거

**날짜**: 2026-05-25
**상태**: Accepted

## 결정

뉴스 청크 생성 시 회사 매칭 (Aho-Corasick 룰 + LLM 3차) 단계를 **완전 제거**.
모든 뉴스를 `corp_code='00000000'` (MACRO) 의 **1 뉴스 = 1 청크** 로 임베딩.

회사별 검색은 **벡터 의미 매칭** 으로 검색 시점에 수행.

## 대안 (기각)

| 대안 | 기각 이유 |
|---|---|
| 룰 매칭 (Aho-Corasick) 유지 | "동부그룹 자회사", "HBM 공급사" 같은 암시 못 잡음. 신규 회사 추가 시 기존 뉴스 재매칭 필요 (룰 갱신) |
| LLM 매칭 (3차 hybrid) | 청크당 3.6초 LLM 비용. 신규 회사 추가 시 6분 재처리 |
| `rule_recheck` 마커 (옛 0.2.0 정책) | 룰만 재실행해 LLM 결과 보존하지만, 결국 검색 시 의미 매칭이 더 robust |

## 근거

- **검색 시 벡터 의미 매칭이 더 robust** — 회사명 직접 언급 없어도 "삼성전자 메모리 사업" 같은 쿼리가 의미적으로 잡음
- **신규 회사 추가 무비용** — `.env CORPS` 추가 후 뉴스 재매칭 단계 없음. 검색 쿼리에 새 회사명 넣으면 끝
- **인덱스 단순화** — 회사별 멀티 청크 (1뉴스 × N매칭) → 1뉴스=1청크로 정합 명확

## 영향 받는 코드

- `chunk/news.py` — `build_chunks()` 단순화 (matched 무관)
- `cli.py build` — `_detect_new_corps_and_mark_rule_recheck` 호출 제거
- `stage_b4_news.py`, `news_matching.py` — 파일 유지 (향후 매칭 부활 시 재사용)

## 트레이드오프

- ❌ 잃는 것: 룰로 정확히 잡힌 회사명 매칭의 precision (예: "삼성전자" 명시 기사)
- ✅ 얻는 것: 룰이 못 잡는 암시 매칭, 신규 회사 추가 무비용, 인덱스 단순
- 골드셋 평가 필요 — 신규 정책의 뉴스 검색 정확도 측정은 후속 작업
