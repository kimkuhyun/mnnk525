# ADR 008 — 의미 그래프 LLM 추출 격리 (`:LLMExtracted` 라벨)

**날짜**: 2026-05-26
**상태**: Accepted

## 결정

LLM (qwen3.5:9b) 으로 추출한 모든 노드·엣지에 Neo4j 보조 라벨 `:LLMExtracted` 부착.
정형 그래프 (DART API 결정론 추출) 와 **라벨로 명시 격리**.

정형 평가 Cypher 는 `WHERE NOT n:LLMExtracted` 로 LLM 추출분 자동 제외.

## 격리 규칙

| 출처 | 라벨 | extracted_by 속성 |
|---|---|---|
| 정형 (DART OpenAPI JSON 결정론 파싱) | 라벨 면제 | `deterministic_rule_v1` |
| 사전 매칭 (organizations.yml 등 alias hit) | 라벨 면제 | `alias_dict_v1` |
| LLM 추출 (Statement/Event/Relation, Product/Tech) | `:LLMExtracted` | `qwen3.5:9b` |
| Vector ER 링킹 (Stage 2) | `:LLMExtracted` (Organization 보강 시) | `qwen3.5:9b` |

## 근거

1. **정형 그래프 무결성 보호** — Cypher 평가(`polaris graph-eval`) 가 결정론 영역만 측정. F1=1.0 회귀 방지.
2. **신뢰도 분리** — LLM 추출은 항상 confidence < 1.0. 사용자/검색 응답에서 명시 가능.
3. **롤백 안전** — `MATCH (n:LLMExtracted) DETACH DELETE n` 한 줄로 LLM 영역 통째 제거 가능 (정형 보존).
4. **혼합 라벨** — Organization 같은 전역 entity 는 사전 매칭 시 라벨 없음, vector ER 으로 신설 시 `:LLMExtracted` 추가. 둘 다 가능.

## Cypher 패턴 예

**정형만 (LLM 제외):**
```cypher
MATCH (o:Organization)-[:HAS_METRIC]->(m:FinMetric)
WHERE NOT 'LLMExtracted' IN labels(o)
RETURN o, m
```

**LLM 포함 (의미 그래프 통합):**
```cypher
MATCH (c:Chunk {chunk_id: $cid})-[:hasObject]->(p:Product)
RETURN p.name, 'LLMExtracted' IN labels(p) AS is_llm
```

**LLM 롤백:**
```cypher
MATCH (n:LLMExtracted) DETACH DELETE n
```

## 영향 받는 코드

- `src/polaris/db/init_neo4j.py` — `:LLMExtracted` 인덱스 (`extracted_by`, `confidence`)
- `src/polaris/graph/merger.py` — `CQ_PRODUCT_LLM_LABEL`, `CQ_TECH_LLM_LABEL`, Statement/Event/Relation 은 항상 `:LLMExtracted` 부착
- `src/polaris/db/graph_diag.py` — C-12 `:LLMExtracted` vs 정형 충돌 체크
- 정형 평가 (`src/polaris/eval/gold_graph.py`) — Cypher 에 `WHERE NOT n:LLMExtracted` 명시 권장

## 트레이드오프

- ✅ 격리 명확, 롤백 1줄
- ✅ 정형 F1=1.0 게이트 영향 X
- ⚠️ Organization 같은 entity 가 사전·LLM 양쪽으로 보강되면 라벨 혼재 — confidence 평균 또는 우선순위 룰 추후 명시
