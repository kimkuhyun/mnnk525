# POLARIS 디비설계 (DB Schema — Source of Truth)

> **이 문서가 DB 스키마의 단일 진실(Source of Truth)이다.**
> 테이블·노드·엣지를 추가/변경하면 **이 문서를 먼저 갱신**하고 `pola/src/polaris/db/init_mariadb.sql` 에 반영한다.
> 기준: 실제 DB 감사 (2026-05-30). 설계 의도·서사·확장 로드맵·정리 대상(폐기 테이블 등)은 → [`pola/docs/ERD.md`](../../pola/docs/ERD.md).

**3-DB 분담**
| DB | 역할 | 포트(dev) |
|---|---|---|
| **MariaDB** | 관계형·집계·원본 SSOT | 3307 |
| **Neo4j** | 그래프·관계·traversal | 7687 |
| **Qdrant** | 벡터·의미검색 (1024d bge-m3, cosine) | 6333 |

## 0. 교차 키 (3-DB 공통 식별자)

| 키 | 형식 | MariaDB | Qdrant payload | Neo4j |
|---|---|---|---|---|
| `corp_code` | 8자리 | 대부분 테이블 | 필수 | `Organization.corp_code` PK |
| `rcept_no` | 14자리(공시) | `document_index`·`dart_raw_index` | 필수 | `FilingDocument.rcept_no` PK |
| `chunk_id` | 16 hex | `chunk_index.chunk_id` PK | 필수 | `Chunk.chunk_id` PK |
| `doc_id` | sha(32) | `document_unified.doc_id` PK | `polaris-doc-*` | `Document.doc_id` |
| `run_id` | `YYYYMMDD_HHMM_NN` | `active_run_manifest` | 필수 | `Chunk.run_id` PK |

> 단일 시드(삼성전자 `00126380`)→다회사 확장은 `corp_code` 가 모든 곳에 있어 **스키마 변경 없이 데이터만 추가**.

## 1. MariaDB — 테이블 (15, init_mariadb.sql 정의)

DDL 원본은 `pola/src/polaris/db/init_mariadb.sql` (idempotent, `IF NOT EXISTS`). 아래는 PK·핵심 컬럼·용도 요약.

### 원본·통합 레이어
| 테이블 | PK | 핵심 컬럼 | 용도 |
|---|---|---|---|
| `dart_raw_index` | (corp_code, endpoint, hash8, run_id) | `rcept_no`·`body_json`(LONGTEXT)·`raw_path`·`status` | DART API 정형 JSON 원본 (SSOT) |
| `news_raw` | news_id | `feed_id`·`publisher`·`title`·`url`·`body`(LONGTEXT)·`meta`(JSON) | 뉴스 원문 (SSOT) |
| **`document_unified`** | doc_id | `source_type`(news\|dart\|ir\|…)·`origin_id`·`corp_code`·`ts`·`title`·`body`·`metadata`(JSON)·`credibility` | **통합 문서 레이어 (CORE).** 모든 근거 doc 의 단일 인터페이스. BE `/evidence`·`/node-evidence`·`/node` |

### 청크·검색 (RAG)
| 테이블 | PK | 핵심 컬럼 | 용도 |
|---|---|---|---|
| `chunk_index` | (chunk_id, run_id) | `corp_code`·`rcept_no`·`chunk_type`(table_nl\|text_micro\|text_macro)·`section_path`·`embedding_text`·`ingest_status` | Qdrant point ↔ 본문 join 진입점 |
| `chunk_summary` | (chunk_id, run_id) | `summary`·`summary_method`·`summary_version` | 청크 LLM 요약 sidecar |
| `document_index` | (rcept_no, run_id) | `corp_code`·`doc_type`·`date`·`title`·`summary_short`·`key_facts`(JSON)·`page_index`(JSON) | 공시 메타 + 요약 (Neo4j FilingDocument 미러) |

### 운영 (Blue/Green)
| 테이블 | PK | 핵심 컬럼 | 용도 |
|---|---|---|---|
| `active_run_manifest` | id(=1, 단일 행) | `active_run_id`·`active_qdrant_collection`·`standby_*`·`standby_status`(enum) | 검색 active / 인입 standby 슬롯 포인터 |

### 집계·서빙 (대시보드 precompute — BE 직빵)
| 테이블 | PK | 핵심 컬럼 | 서빙 API |
|---|---|---|---|
| `news_matched` | (news_id, run_id) | `matched_corps`(JSON)·`rule_hits`·`llm_hits`·`method` | (pola 매칭 산출) |
| `mention_daily` | (corp_code, date, source_type) | `mention_cnt`·`evidence_doc_ids`(JSON) | `/trend` |
| `sentiment_daily` | (corp_code, date) | `pos`·`neg`·`neu` | `/sentiment` |
| `doc_sentiment` | doc_id | `corp_code`·`label` | (파이프라인 전용, API 미노출 → sentiment_daily 집계 원천) |
| `news_daily_summary` | (corp_code, date) | `summary`·`article_count` | `/daily-digest`·`/briefing` |
| `stock_daily` | (corp_code, date) | `close`·`change_pct`·`volume` | `/stock` (워터마크 증분) |
| `edge_snapshot` | id(auto) | `run_ts`·`corp_code`·`grp`·`predicate`·`target` | `/changes` (엣지 소멸 비교) |
| `ir_report` | rcept_no | `corp_code`·`doc_type`·`date`·`title`·`raw_text`·`summary` | `/ir-reports` (※ deviation — pola ERD 참조) |

**핵심 흐름**
```
원본(news_raw / dart_raw_index) → document_unified(통합) → chunk_index → Qdrant 임베딩
                                       ↘ 집계: mention_daily / sentiment_daily / news_daily_summary
DART 정형 → Neo4j FinMetric · INVESTS_IN 등
```

## 2. Neo4j — 노드 라벨 · 관계

### 노드 라벨 (주요)
| 라벨 | PK / 식별 | 비고 |
|---|---|---|
| `Organization` | corp_code (8자리) | 회사 (DART) |
| `Company` | corp_code | ⚠ `Organization` 과 **함께 부여되는 이중(co-)라벨**(load_neo4j). 일부 BE 쿼리가 의존 → 유지하되 중복 정리 검토(pola ERD §9.2) |
| `NewsEntity` | ext_id | 뉴스 추출 엔티티 (corp_code 없음) |
| `Person` | person_id | 임원·주주 |
| `FilingDocument` | rcept_no | 공시 메타 |
| `Document` / `NewsArticle` | doc_id | v2 문서 레이어 (`document_unified` 미러) |
| `Chunk` | (chunk_id, run_id) | 임베딩 단위 |
| `FinMetric` | metric_id | 재무지표 (account_id·year·reprt_code·fs_div) |
| `Product` / `Technology` | product_id / tech_id | 제품·기술 |
| `BusinessGroup` | unityGrupCode | 기업집단 (FTC) |
| `MacroIndicator` / `StatTable` | stat_code / list_id | 거시지표·통계표 |
| `Statement` / `Event` | (id, run_id) | claude-direct 추출 |
| `LLMExtracted` | (marker) | 추출 노드 식별용 보조 라벨 |

### 관계 (그룹별)
- **정형(DART, `extracted_by` IS NULL = 공시·사실)**:
  `HAS_METRIC`(Org→FinMetric) · `INVESTS_IN` · `IS_MAJOR_SHAREHOLDER_OF` · `AFFILIATED_WITH`(Org→BusinessGroup) · `EXECUTIVE_OF`(Person→Org) · `IS_SUBSIDIARY_OF` · `DERIVED_FROM`(FinMetric→FilingDocument)
- **뉴스/문서**: `MENTIONS` · `ABOUT`(Document→Org) · `hasActor`
- **관계 인텔리전스(claude, `extracted_by='claude'` = 뉴스 언급)**:
  `PARTNERS_WITH` · `SUPPLIES` · `COMPETES_WITH` · `CUSTOMER_OF` · `LITIGATION` · `ACQUIRES` · `LICENSES` · `JV_WITH`
- **기타**: `DEVELOPS`(Org→Product/Tech) · `wasDerivedFrom`(PROV)

**엣지 출처 규약(필수 유지)**: `r.extracted_by='claude'`(뉴스 언급) vs `IS NULL`(DART 공시·사실).
- 뉴스 엣지 속성: `doc_ids`·`evidence_count`
- DART 엣지 속성: `qota_rt`(지분)·`rcept_no`·`purps`(목적)·`amount`·`first_acq`(취득일)

## 4. 변경 규칙 (요약)

1. 새 테이블/라벨/엣지 = **이 문서 먼저 갱신** + `init_mariadb.sql` 반영. 모듈 `ensure_tables` 만으로 끝내지 말 것.
2. 문서형 데이터(뉴스/공시/IR/특허 등)는 `document_unified`(source_type) 경유. 독립 doc 테이블 신설 금지 — 불가피하면 pola ERD 에 deviation 기록.
3. 교차키(`corp_code`/`rcept_no`/`chunk_id`/`doc_id`/`run_id`) 일관 유지.
4. 엣지는 출처 구분(`extracted_by`) 유지.

> 상세 변경 규칙·폐기 대상·deviation·확장 로드맵 → [`pola/docs/ERD.md`](../../pola/docs/ERD.md).
